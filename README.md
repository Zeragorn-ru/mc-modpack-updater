# Dustpack Updater

Небольшой консольный апдейтер на Kotlin для автоматического обновления модпака **Dustpack** через GitHub Releases.  
Предназначен для запуска из **PrismLauncher** и аналогичных лаунчеров.

---

## Возможности

- Получает последний релиз с GitHub
- Работает через `digest` (SHA-256), не качает файлы без изменений
- Поддерживает `removes.txt` для точечного обновления файлов
- Не перезаписывает существующие файлы при распаковке
- Может обновлять **сам себя**
- Весь конфиг хранится в одном `updater-config.json`
- Собирается в один `fat jar` со всеми зависимостями

---

## Как это работает

### 1. Архив (`update.zip`)
- Ищется в assets последнего релиза GitHub
- Если `digest` изменился:
  - архив скачивается
  - распаковывается в корень инстанса
  - **существующие файлы не заменяются**
- Если файл нужно обновить — он должен быть указан в `removes.txt`

---

### 2. removes.txt
- Также берётся из assets релиза
- Если `digest` изменился:
  - файл скачивается
  - перечисленные в нём файлы удаляются
  - `removes.txt` удаляется после выполнения
- Может содержать **любой файл**, включая сам updater

---

### 3. Конфиг

Файл `updater-config.json` создаётся автоматически и содержит:

```json
{
  "archiveDigest": "sha256:...",
  "removesDigest": "sha256:..."
}
```

Используется для определения, изменились ли файлы с прошлого запуска.

---

## Важно про директории

Updater **всегда работает относительно `../`**, а не текущей директории запуска.

Это сделано специально, потому что:
- PrismLauncher запускает процессы из своей runtime-папки
- Корень инстанса находится уровнем выше

Никаких аргументов при запуске не требуется.

## Сборка

### Требования
- Java 21
- Gradle (через wrapper)

### Сборка fat jar

```bash
./gradlew clean fatJar
```

Готовый файл появится здесь:

```
build/libs/updater-all.jar
```

---

## Запуск

```bash
java -jar updater-all.jar
```

Можно запускать:
- вручную
- через PrismLauncher
- как часть startup-скрипта

---

## Поведение при ошибках

- Если GitHub API недоступен — updater завершится без изменений
- Если файл уже существует — он **не будет перезаписан**
- Удаление собственного `.jar` **не мешает текущему выполнению**
